06 · Remediation addendum

Seven fixes for the live app.

Issues pulled directly from your screenshots of the running CardScout web app. Each has a specific diff against token values, a shipping severity, and a one-line intent for why it matters. Ordered by impact: fix S1 and S2 first for the biggest quality-perception lift.

01
severity · high

Three different dark modes are shipping

Homepage dark ≈ slate-blue, Tax guide dark ≈ near-black, wizard result dark ≈ blue-tinted. REDESIGN_TRACKER specified #111318 — it never appears. Unify on one canvas; collapse the others.

Before
Home: --bg: #0E1320
Learn: --bg: #0A0F1C
Wizard: --bg: #13182A
After
--bg: #111318
--surface: #1A1E27
--surface-raised: #232A37
02
severity · high

Emoji in production UI

Card Finder wizard: 💰 Cashback · ✈️ Travel rewards · 🎁 Signup bonus · 🧾 Everyday spending. Signup Bonus Planner: same pattern. Swap to Lucide icons — single biggest perceived-quality lift available in one pass.

Before
💰 Cashback
✈️ Travel rewards
🎁 Signup bonus
🧾 Everyday spending
After
<Coins/> Cashback
<Plane/> Travel rewards
<Gift/> Signup bonus
<Wallet/> Everyday spending
03
severity · medium

Dark-mode hero blue fails WCAG contrast

"We'll find your best card" renders #1B4FD8 on #0E1320 — contrast ratio ≈ 3.8, below AA 4.5. REDESIGN_TRACKER already specified a dark-mode lift; wire it up.

Before
[data-theme="dark"]
--accent: #1B4FD8
contrast 3.8 · fail
After
[data-theme="dark"]
--accent: #4F7FFF
contrast 6.2 · AA pass
04
severity · medium

Left-border accents are semantically confused

Chase = blue, Venture X = red, Amex Gold = green, Discover = orange. Reads as "color per card" but isn't a system — it creates random color noise. Recast left-border as rank + urgency; move issuer color into the card artwork where it belongs.

Before
border-left: per-issuer
Chase → blue · AmEx → green
Cap One → red · Disc → orange
After
border-left: per-rank
Top match → accent
Runner-up → border
Urgent / warning → urgent
05
severity · high

Results page still ships 3-column, not v4.0 list-item

Current: 3 product cards side-by-side, 3 "Apply Now" CTAs visible at once — reads ad-like and buries the recommendation. v4.0 spec called for list-item with inline expand on the top match, collapsed rows for alternates. See §3 above for the new component.

Before
grid-template-columns:
repeat(3, 1fr)
3 × Apply Now buttons
density: ad-like
After
display: flex; flex-dir: column
Top match → expanded + CTA
Alternates → collapsed rows
density: editorial
06
severity · low · style consistency

Instrument Serif inconsistent across H1s

Home hero uses it ✓. Tax Payments guide H1 uses Manrope bold ✗. Card Planner "Which of these sounds like you?" uses Manrope bold ✗. Decision (made): Instrument Serif italic on all destination H1s (Home, wizard results, Vault dashboard, Benefit Detail). Learn/guide articles stay on Manrope — editorial long-form reads better sans.

After

Guides H1: Manrope 800 (keep)
